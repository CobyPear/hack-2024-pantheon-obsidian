• 1. Install the plugin
  • 2. Setup

    • Requirements:

    • A Pantheon account and a workspace with Front-End Sites enabled
    • A GitHub account

    Steps

    After installing the plugin, some additional setup is required before publishing can be done.

    1. Create a GitHub repo. It should be public and empty!
    2. In Obsidian.md, open Settings > Community Plugins and click the gear icon on the Hackathon 2024 Pantheon Obsidian Plugin
    3. Configure your GitHub username and repo on the settings page
    4. Hit Publish! This will take your vault content that is NOT marked with a hashtag "draft", convert it to HTML, and push it to the configured GitHub repo. Ensure your local git user is configured properly. If you have trouble with this step reach out!
    5. On the Pantheon dashboard, create a new Front-End Site with the repo configured for the vault. Use the Import Repository flow and select Gatsby as the framework. The rest of the options should be fine as the default.
    6. After the build and deploy steps your site should be available to view.
    7. Write or change content and hit publish to trigger new builds to the site. No more manual configuration should be necessary.
